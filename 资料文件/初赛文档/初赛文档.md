+ 项目研发情况
  + 本项目基于 RISC-V 指令集架构，完成了一款支持五级流水线的多周期 CPU 设计与实现。项目整体按照取指（IF）、译码/寄存器读取（ID）、执行（EX）、访存（MEM）、写回（WB）五个阶段进行模块化开发，采用 Verilog HDL 完成硬件描述，并基于本地Vivado平台进行功能仿真与综合验证,并在服务器上实现仿真验证。
  + 在研发过程中，首先完成了 CPU 顶层架构的规划，随后逐步实现 PC、指令存储器（IM）、数据存储器（DM）、寄存器堆（RF）、算术逻辑单元（ALU）、立即数扩展模块（EXT）、控制单元（ControlUnit）以及流水线数据打拍寄存器等核心模块。之后进一步实现了五级流水线的数据通路连接与控制信号传递。
  + 在流水线设计过程中，重点解决了流水线中的数据冒险与控制冒险问题。针对常见数据冒险相关问题，实现了 ALU 结果前递机制，用于解决 EX 阶段与 MEM 阶段的数据依赖；对于无法通过前递解决的 load-use hazard，则采用暂停流水线与插入气泡的方法进行处理。同时，对于 beq、bne、jal 与 jalr 等跳转类指令，在 EX 阶段完成跳转判断，并通过 flush 机制冲刷流水线中错误取入的指令，从而保证程序执行正确性。
  + 项目开发过程中，采用模块化、分层化设计方法，各功能模块均进行了独立仿真测试，并编写了多组测试指令验证 CPU 功能正确性，包括算术运算、逻辑运算、访存操作、跳转操作以及流水线冲突测试等内容。
+ 设计方案概要
  + 本设计采用经典五级流水线结构，整体包括以下五个阶段

| 流水阶段                                | 功能说明                           |
| --------------------------------------- | ---------------------------------- |
| IF（Instruction Fetch）                 | 根据 PC 从指令存储器中读取指令     |
| ID（Instruction Decode/Register Fetch） | 指令译码、寄存器读取、控制信号生成 |
| EX（Execute）                           | ALU 运算、地址计算、分支判断       |
| MEM（Memory Access）                    | 数据存储器访问                     |
| WB（Write Back）                        | 运算结果写回寄存器堆               |

CPU 支持题目要求的 16 条 RISC-V 指令，包括：

- 算术运算指令：add、sub、addi
- 逻辑运算指令：and、or、xor、ori
- 移位指令：sll、srl、sra
- 访存指令：lw、sw
- 跳转分支指令：beq、bne、jal、jalr

系统支持以下寻址方式：

- 寄存器寻址
- 立即数寻址
- 基址寻址
- PC 相对寻址
- 伪直接寻址

在数据通路设计中，CPU 采用统一的模块化结构，通过多路选择器（MUX）实现不同数据源切换，并通过控制单元生成各阶段所需控制信号。

+++

+ 为提升流水线执行效率，设计中实现了以下流水线优化机制：
  + 数据前递
    + 针对 EX/MEM 阶段产生的数据相关，采用 ALU 结果直接前递到 EX 阶段输入的方法，减少流水线暂停次数。
  + 流水线暂停
    + 针对 lw 指令导致的数据尚未返回的问题，以及部分无法通过前递解决的冒险情况，采用冻结 PC 与 IR 的方式暂停流水线，并插入气泡指令保证执行正确性。
  + 流水线冲刷
    + 对于 jal、jalr、beq、bne 等跳转类指令，在 EX 阶段完成跳转判断后，对错误进入流水线的指令进行冲刷处理。
