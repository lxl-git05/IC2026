# 技术文档

## 1. 模块功能介绍

### 1-1 PC模块

​	PC（Program Counter）模块用于保存当前指令地址，是处理器取指阶段（IF）的核心模块，负责控制指令执行流程的推进。

​	该模块在时钟上升沿接收由NPC模块计算出的下一条指令地址 `NPC`，并在写使能信号 `PCWrite` 有效时完成更新，实现程序的顺序执行与跳转执行。当执行普通指令时，PC按顺序更新；当执行分支或跳转指令时，PC更新为目标地址，从而改变程序执行流向。

​	同时，为支持流水线中的暂停机制，当检测到数据冒险时，控制单元可通过拉低 `PCWrite` 使PC保持当前值不变，暂停取指阶段，避免错误指令进入流水线。

​	此外，模块支持异步复位功能。复位时，PC被初始化为 `0xFFFF_FFFC`，保证下一次更新后程序从地址 `0x0000_0000` 开始执行。

​	PC模块作为CPU控制流的起点，承担了程序地址保存、顺序推进、跳转控制以及流水线暂停控制等关键功能。

### 1-2 IM模块

​	IM（Instruction Memory）模块用于存储处理器执行的程序指令，是五级流水线中取指阶段（IF）的核心组成部分。

​	该模块根据程序计数器（PC）提供的地址信号 `addr`，从指令存储器中读取对应的32位机器指令，并输出到指令寄存器（IR），供后续译码阶段使用。模块内部采用存储数组 `memory[0:1023]` 作为指令存储空间，可支持最多1024条指令存储。

​	本模块采用组合逻辑设计，当地址或读使能信号 `InsMemRW` 发生变化时，立即完成指令读取，提高取指效率，满足流水线连续取指需求。

​	控制信号 `InsMemRW` 用于控制指令存储器的读取操作，在正常运行时始终保持使能状态，以保证处理器每个周期能够完成取指操作。

​	IM模块主要承担程序指令存储与读取功能，是整个CPU执行流程的数据入口模块。

### 1-3 ControlUnit模块

​	ControlUnit模块是CPU的核心控制模块，负责对指令进行译码，并生成各阶段所需的控制信号，协调整个五级流水线的数据流与控制流。

​	该模块根据输入指令的 `opcode`、`Funct3` 和 `Funct7` 判断指令类型，支持R型、I型、Load、Store、Branch、JAL和JALR等多种指令格式，并生成对应的控制信号，包括寄存器写使能、存储器写使能、ALU运算控制、下一条PC选择以及写回数据来源选择等。

​	在流水线执行过程中，ControlUnit内部对控制信号进行分阶段寄存（ID→EX→MEM→WB），保证控制信号与数据在流水线中的同步传递。

​	同时，该模块实现了流水线冒险处理机制：

1. **数据前递机制（Forwarding）**
    支持EX阶段和MEM阶段结果前递到当前EX阶段，解决大部分RAW数据冒险。
2. **加载-使用冒险检测（Load-Use Hazard）**
    当检测到lw指令结果被下一条指令立即使用时，暂停流水线一个周期，并插入气泡。
3. **控制冒险处理（Flush）**
    对beq、bne、jal和jalr跳转指令进行冲刷处理，清除错误进入流水线的指令。

​	ControlUnit模块是整个CPU的控制中心，承担指令译码、控制信号生成、流水线控制以及冒险处理等关键功能。

### 1-4 EXT模块

​	EXT（Extend）模块用于将指令中的12位立即数扩展为32位数据，供ALU进行运算使用。

​	该模块根据控制信号 `ExtSel` 选择扩展方式：

- **零扩展（Zero Extension）**：用于逻辑立即数指令（如ori）
- **符号扩展（Sign Extension）**：用于算术立即数指令（如addi、lw、sw、jalr）

​	通过立即数扩展，EXT模块保证不同类型指令能够正确生成运算操作数。

​	EXT模块主要承担立即数格式转换功能，是指令执行阶段的重要辅助模块。

### 1-5 IR模块

​	IR（Instruction Register）模块用于保存取指阶段获取的指令，实现IF阶段与ID阶段之间的数据传递。

​	该模块在时钟上升沿将指令存储器输出的指令锁存到内部寄存器中，并输出给控制单元和寄存器堆进行译码处理。

​	当流水线发生暂停时，IR通过 `IRWrite` 控制保持当前指令不变，实现流水线冻结。

​	当发生跳转或分支成功时，IR通过 `flush` 信号写入NOP指令，完成流水线冲刷，避免错误指令继续执行。

​	IR模块承担流水线级间指令缓存、暂停保持和冲刷控制功能，是流水线控制的重要组成部分。

### 1-6 NPC模块

​	NPC（Next Program Counter）模块用于计算下一条指令地址，是CPU控制流跳转的核心模块。

​	该模块根据控制信号 `NPCOp` 选择不同的地址更新方式，实现顺序执行、条件跳转和无条件跳转功能。

​	支持的地址更新方式包括：

- **顺序执行（PC+4）**：用于普通指令连续执行；
- **分支跳转（beq、bne）**：根据分支偏移量计算目标地址；
- **无条件跳转（jal）**：根据J型偏移量生成跳转目标地址；
- **寄存器跳转（jalr）**：根据寄存器值与立即数计算跳转地址。

​	为了适配五级流水线结构，NPC模块内部对PC和跳转偏移量进行流水线寄存，使跳转目标地址能够在EX阶段正确生成。同时保存 `PC+4` 并传递到WB阶段，用于jal和jalr指令的返回地址写回。

​	此外，为解决jalr指令中的数据冒险问题，模块引入前递后的最新寄存器值 `rs_A` 参与地址计算，实现数据前递，保证跳转地址正确。

​	NPC模块承担程序流控制、跳转地址生成以及返回地址保存等关键功能。

### 1-7 MUX_2to1_A模块

MUX_2to1_A模块用于选择ALU第一个操作数，是执行阶段（EX）的输入选择模块。

该模块根据控制信号选择不同的数据源作为ALU输入，包括：

- 寄存器读出数据；
- 固定常数0；
- EX阶段前递结果；
- MEM阶段前递结果。

通过数据选择功能，该模块实现了流水线中的数据前递机制，解决指令之间的数据相关问题，提高流水线执行效率。

MUX_2to1_A模块主要承担ALU输入选择与数据前递功能。

### 1-8 MUX_3to1_B模块

MUX_3to1_B模块用于选择ALU第二个操作数，是执行阶段（EX）的重要数据选择模块。

该模块根据控制信号选择不同输入来源，包括：

- 寄存器读出数据；
- 扩展后的立即数；
- 偏移地址；
- EX阶段前递结果；
- MEM阶段前递结果。

模块支持算术运算、访存地址计算以及分支地址计算等多种场景，并结合数据前递机制解决流水线数据冒险问题。

MUX_3to1_B模块承担ALU第二操作数生成与前递选择功能。

### 1-9 ALU模块

ALU（Arithmetic Logic Unit）模块是CPU执行阶段（EX）的核心运算模块，负责完成算术运算、逻辑运算和比较操作。

该模块根据控制单元生成的 `ALUOp` 控制信号执行对应操作，支持以下运算：

- 加法（add、addi、lw、sw、jalr）
- 减法（sub、beq、bne）
- 按位与（and）
- 按位或（or、ori）
- 按位异或（xor）
- 逻辑左移（sll）
- 逻辑右移（srl）
- 算术右移（sra）
- 有符号比较（slt）
- 无符号比较（sltu）

同时，模块输出比较结果标志位 `zero`，用于分支指令判断是否满足跳转条件。

ALU模块承担数据运算与条件判断功能，是CPU执行阶段的核心计算单元。

### 1-10 DM模块

DM（Data Memory）模块用于实现CPU的访存功能，负责数据的读写操作，是Load/Store指令执行阶段（MEM）的核心模块。

该模块内部包含一个32位宽的数据存储器 `memory[0:1023]`，用于存储程序运行过程中的数据。

DM模块支持两种操作模式：

- **写操作（Store）**：当控制信号 `DMCtrl` 为1时，在时钟上升沿将写数据 `WD` 写入指定地址；
- **读操作（Load）**：当 `DMCtrl` 为0时，根据地址 `Addr` 异步读取数据并输出。

为了保证流水线数据一致性，写入数据在写入前进行寄存处理（打一拍），避免组合逻辑不稳定带来的错误写入。

DM模块采用“同步写、异步读”结构，既保证写入可靠性，又提高读取效率，是典型的流水线访存结构设计。

该模块主要承担数据存储、数据读取以及Load/Store指令执行功能。

### 1-11 MUX_3to1_LMD模块

MUX_3to1_LMD模块用于在写回阶段（WB）选择最终写入寄存器的数据来源，是流水线写回路径的关键选择单元。

该模块根据控制信号 `WDSel` 在以下三种数据来源中进行选择：

- **ALU结果（FromALU）**：算术逻辑运算结果；
- **数据存储器结果（FromMEM）**：Load指令读取的数据；
- **PC+4（FromPC）**：用于jal/jalr指令的返回地址写回。

其中，ALU结果在写回前进行寄存处理（打一拍），保证流水线数据同步；PC+4由NPC模块提供，用于实现函数调用返回地址保存。

该模块实现了不同指令写回数据的统一选择机制，是寄存器写回阶段的重要数据通路控制模块。

### 1-12 Flopr模块

​	Flopr本质是一个带复位的32位寄存器，用于在时钟上升沿锁存数据，实现流水线各级之间的“拍间隔离”。复位时输出清零，正常情况下 `out_data <= in_data`。在本设计中，它被用于RD1/RD2、ALU结果、数据存储器输出等关键路径，避免组合逻辑串联导致时序过长，是数据通路稳定性的基础组件。

### 1-13 IR模块

​	IR用于保存IF阶段取出的指令，并支持流水线冲刷机制。当 `IRWrite=1` 时更新指令；当 `flush=1` 时，不写入真实指令而是强制写入 `32'h00000013`（NOP），用于消除控制冒险带来的错误指令影响。该机制在BEQ/BNE成立或JAL/JALR跳转时触发，使错误路径指令在进入ID阶段前被清除。

### 1-14 打拍延迟模块

​	由于顶层需要保持clean, 所以在多个模块内部加入信号延迟打拍模块，实现信号的延迟输入或输出。

## 3. 功能划分

​	本设计采用经典五级流水线结构，将指令执行过程拆分为 IF、ID、EX、MEM、WB 五个阶段，各阶段职责明确，时序和打拍进行隔离，与控制信号传递实现并行执行。

### IF（取指阶段）

该阶段由PC与指令存储器共同完成，核心任务是产生下一条指令地址并读取指令。

PC负责在时钟上升沿更新PC值，当 `PCWrite=1` 时执行 `PC <= NPC`，否则保持不变，用于支持stall控制。

IM根据PC地址读取指令 `Ins`，为组合逻辑结构，输出随地址变化即时更新。

该阶段输出：

- 当前指令地址 PC
- 下一条指令 PC+4（作为默认顺序执行路径）

------

### ID（译码/寄存器读取阶段）

该阶段负责指令解析与寄存器读取，同时生成控制信号。

主要功能包括：

- 从指令中解析 opcode / rs1 / rs2 / rd / immediate
- 读取寄存器堆数据 RD1、RD2
- 产生控制信号（ALUOp、ALUSrc、RegWrite等）
- 执行数据前递选择（Forwarding判断）

ControlUnit在该阶段生成绝大多数控制信号，并同时检测数据冲突（load-use、store hazard、jalr hazard）以及控制冲突（branch/jump）。

------

### EX（执行阶段）

该阶段完成算术逻辑运算与分支目标计算，是数据前递的核心阶段。

功能包括：

- ALU执行（加减、逻辑运算、移位、比较）
- 分支条件判断（zero信号）
- 计算跳转目标地址（branch/jal/jalr）
- 前递数据参与运算（Forwarding A/B）

ALU执行所有算术与逻辑运算，并输出 zero 信号用于分支判断。

NPC根据 NPCOp 选择 PC+4、branch偏移、jal偏移或jalr地址，决定程序流向。

该阶段是：

- 数据冒险解决的关键（forwarding）
- 控制冒险决策点（branch/jump）

------

### MEM（访存阶段）

该阶段负责数据存储器访问。

DM在该阶段根据 ALU 计算出的地址进行读写操作：

- Store：写入内存
- Load：读取数据

该阶段特点：

- 同步写（posedge clk）
- 异步读（组合输出）
- 支持 load-use hazard 检测来源

------

### WB（写回阶段）

该阶段将最终结果写回寄存器堆。

MUX_3to1_LMD用于选择写回数据来源：

- ALU结果
- MEM读取数据
- PC+4（用于JAL/JALR）

RF在该阶段完成寄存器写回操作。
